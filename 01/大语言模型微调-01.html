Chapter 01
CHAPTER ONE

大语言模型的本质

大型语言模型是一个运行在 GPU 上的、极其庞大且复杂的确定性数学函数。它的核心任务只有一个：接收数字化的文本输入，经过数以百亿计的矩阵乘法和加法运算，最终在输出端生成一个概率分布，预测下一个最有可能出现的词汇。

我们将大模型的运行机制拆解为五个连续的物理与计算步骤：

输入的数字化：分词与词表

1. 词表的构建

在模型被训练之前，开发人员会为模型定义一个固定的"词表"，其中每一个基础文本单元称为 Token。主流大模型的词表大小通常在 3 万到 20 万个 Token 之间，每个 Token 有唯一整数 ID。

2. 分词的过程

分词器通过字符串匹配算法，将用户输入的长文本切分成一个个 Token，并将其转换为整数 ID，形成一维整数数组。

高维空间映射：词向量

离散整数 ID 之间没有数学关联，无法进行语义计算。因此需要将其转化为高维浮点数数组——词向量。现代大模型的内部空间维度通常在 4096 维到 12288 维之间。

关键洞见：语义相近的词（如"苹果"和"香蕉"），其向量在高维空间中的几何距离很近；语义无关的词（如"苹果"和"挖掘机"）距离则很远。词与词之间的语法关系，以向量差值的方向表示。

🔬 动画演示 · 词向量语义距离

高维词向量降维后的二维投影——语义相近的词在空间中聚集，语义无关的词相距甚远。

余弦相似度：苹果↔香蕉 = 0.91  |  苹果↔挖掘机 = 0.08

参数与矩阵乘法

1. 参数的物理定义

大模型的"参数"是神经网络中的权重（Weights）和偏置（Biases），以大量浮点数值的形式静态存储在显卡内存（VRAM）的矩阵中。模型的知识、逻辑、语法能力，全部以这些浮点数的特定数值组合存储其中。

2. 前向传播：数据的逐层计算

输入文本的向量矩阵进入模型后，会自下而上依次穿过模型的每一个计算层，在每一层与该层的参数矩阵进行密集的矩阵乘法操作，触发注意力机制，逐层提取越来越高级的语义特征。

注意力机制："银行"这个词，在与"河流"共同出现时，其向量会被修改为偏向"河岸"的语义；在与"金融"共同出现时，则偏向"金融机构"的语义。上下文信息通过矩阵运算被注入词向量。

概率预测与输出

经过数十个隐藏层的运算后，最终输出的高维向量会通过输出矩阵压缩到词表大小的维度（如 100000 维），得到对数几率（Logits）。再经过 Softmax 函数转换为严格的概率分布。

自回归生成（Autoregressive Generation）

生成的 Token 被附加到输入序列末尾，再次进行完整的全参数矩阵运算，预测下一个词。这就是大模型文本"逐字蹦出"的根本原因——每输出一个字，都是一次完整的数百亿参数矩阵运算。

预训练

预训练是大模型诞生前最漫长、最昂贵的物理过程。在数千张顶级 GPU 组成的计算集群中，对数十 TB 级别的高质量纯文本数据进行自监督学习。

1
遮挡与预测

从海量文本截取片段（如"地球是太阳系由内及外的第三颗___"），隐藏最后一个词，输入给参数随机的模型。

2
前向计算

模型使用当前随机参数进行海量矩阵运算，给出一个概率分布（此时大概率预测错误）。

3
计算误差 Loss

将预测结果与真实答案对比，用交叉熵损失函数计算出一个标量误差值。误差越大，预测越偏离正确答案。

4
反向传播

通过微积分的链式法则，从输出层逆向计算出每一个参数的"梯度"——即应该增大还是减小的方向与幅度。

5
优化迭代

优化器按照梯度方向，将数百亿个参数的数值稍微修改一个极小的幅度（由学习率控制），模型向"正确"方向微小挪动。

宏观涌现：上述循环在 GPU 集群中以每秒数万次的速度不分昼夜运行数月，最终将人类语言的底层统计规律、语法结构、逻辑推导和世界常识，高度压缩编码进数百亿参数的浮点数值分布中——这就是"预训练基座模型"。

微调的物理过程

预训练基座模型仅是被优化用于"续写文本"的机器——若输入"中国的首都是哪里？"，它大概率不会答"北京"，而是继续输出"日本的首都是哪里？美国的首都是哪里？"。

微调是一个"基于误差反向传播，对模型内部原有参数矩阵进行有针对性的小幅度二次修改"的完整计算工程。

数据的结构化重构与损失掩蔽

1. 结构化数据的拼接与向量化

微调数据通常组织为 JSON 格式的"问答对"，系统使用特殊标记将字段拼接为完整字符串，再经分词器转化为 Token 序列进入模型计算。

2. 损失掩蔽（Loss Masking）

微调的目标是让模型学会"根据问题生成答案"，而非学习如何提问。因此系统为输入矩阵生成一个掩蔽矩阵：对应"问题"部分标记为 0（忽略），对应"答案"部分标记为 1（计算误差）。

点击重播后，蓝色"指令"部分将被遮蔽（Loss=0），只有绿色"答案"部分参与误差计算。

前向传播与强制教导

1. 并行计算与非自回归生成

在微调阶段，由于显卡内存中已有完整的标准答案，系统将整个序列（例如 1024 个 Token 的矩阵）一次性平行输入到模型的数十个隐藏层中，同时进行数万亿次浮点乘加运算，以最大化 GPU 并行效率。

2. 强制教导策略（Teacher Forcing）

模型针对第 N 个位置的预测，其上下文始终是来自数据集的前 N-1 个真实 Token，而非模型自己的预测结果。这从物理上保证了误差不会在序列中累积放大，每个位置都在纯净的真实上下文中进行计算。

计算损失函数

交叉熵损失

将模型预测的概率分布，与标准答案的 One-Hot 编码（目标词概率=1.0，其余=0.0）对比，计算出一个具体的标量误差值。误差越大，参数越偏离期望状态；微调的终极目标是让误差趋近于 0。

误差的反向传播

梯度的数学定义

"梯度"是一个与模型参数矩阵形状完全相同的新矩阵，记录着每一个参数应该增大还是减小、幅度有多大。

链式法则的逐层推导

反向传播从最后一层（输出层）开始，逆着前向传播方向逐层计算。为了准确计算每一层的梯度，显卡必须保留所有中间激活值直到反向传播完成——这就是为什么微调对显存的要求远高于推理。

显存消耗警告：微调时显卡需同时持有：原始参数矩阵 + 梯度矩阵 + 优化器状态矩阵 + 中间激活值。一个 7B 参数的模型推理约需 14GB 显存，而微调则可能需要 60GB+ 显存。

参数的物理更新与迭代

学习率的物理约束

梯度只是局部的切线斜率，不能直接叠加到参数上。必须乘以极小的缩放系数——学习率（Learning Rate）——控制每步更新幅度，防止越过最优解导致模型崩溃。

内存中的数值覆写

优化器执行全矩阵加减法：旧参数 − (梯度 × 学习率) = 新参数，直接覆写显存中的旧数值。模型内部发生了一次永久性的微小改变。

CHAPTER THREE

什么时候需要微调

微调的核心判断标准：我们是希望改变模型输出的"概率分布模式"，还是希望增加模型内部的"静态数据总量"？

微调只能改变行为模式（怎么回答），无法有效注入新事实知识（知道什么）。

适用场景：模式对齐与输出分布重构

场景一：结构化格式的严格对齐

痛点：未经微调的模型即使被要求"只输出 JSON"，其参数计算出的概率分布仍极有可能在开头生成"好的，以下是为您提取的 JSON 格式数据："——哪怕多一个标点，下游代码系统就会崩溃。

微调效果：通过反向传播，系统性地压制自然语言词汇的输出几率，抬升 {、"、[ 等结构化符号的输出几率，从根本上消除格式不确定性。

场景二：特定领域语气、风格与行为规范的固化

痛点：基座模型默认参数学自大量口语化网络文本，可能在专业法律咨询中夹杂"大概"、"我觉得"等不严谨词汇。

微调效果：通过数千条专业律师撰写的对话数据，反向传播精准定位控制词汇选择的参数层，使专业术语的输出几率被永久性调高。无需每次通过冗长提示词约束语气。

场景三：垂直领域固定任务的逻辑强化

痛点：医疗"病历信息抽取"任务中，原始文本冗长混乱，模型的注意力计算资源容易被无关 Token 干扰，导致提取遗漏。

微调效果：模型内部的注意力机制权重矩阵发生物理重组——对无关闲聊赋予极低注意力权重，对医学专业名词和时间节点赋予极高权重。这种"专业且抗干扰的提取逻辑"直接硬编码进参数矩阵。

不适用场景：动态知识的注入与记忆

最常见且最致命的技术误区：企图通过微调让大语言模型"记住"新的事实、新的文章或企业内部数据库。

!
非隔离性存储

大模型的知识弥散且重叠地分布在数百亿参数中，关于"相对论"的知识和关于"苹果派菜谱"的知识，可能共享数百万个相同的权重参数——不存在类似数据库表的物理隔离。

!
灾难性遗忘（Catastrophic Forgetting）

强行将财报数据写入参数时，被修改的参数原本可能负责维持"基础数学加减法"或"通用语法结构"。新知识的强制覆盖直接破坏原有参数的高维空间拓扑结构，导致模型丧失逻辑推理能力甚至输出乱码。

!
算力浪费与幻觉

通过海量矩阵运算将纯文本压缩成几个浮点数的微小改变，数据转化效率极低，通常需要将同一篇文章循环训练数十次。且当被问及未在训练数据中明确写出的细节时，模型极有可能用严谨的语气编造出完全不存在的虚假数字（幻觉）。

替代方案：检索增强生成（RAG）

RAG 的核心物理理念：将"动态的知识存储"与"静态的逻辑推理引擎"在物理上彻底剥离。绝不修改大模型的任何内部参数，而是在输入端进行数据的动态组装。

1
外部知识库的向量化存储

将大量外部文件切割成文本块，使用嵌入模型转换为高维浮点数向量，存储在独立的向量数据库中。

2
用户查询的数学映射与检索

将用户问题转换为向量，在向量数据库中进行余弦相似度的几何数学运算，找出高维空间中距离最近的相关文档块。

3
上下文组装与一次性推理

系统自动将检索到的相关文本与用户问题拼接成巨大的提示词，一次性输入大模型。由于最新知识已作为临时上下文存在于输入端，模型的注意力机制可以精准聚焦于这些文本输出准确答案。

RAG 的物理优势：整个过程中模型权重参数零修改，完全杜绝灾难性遗忘。若知识更新，只需修改向量数据库中的对应条目，无需重新训练大模型。
