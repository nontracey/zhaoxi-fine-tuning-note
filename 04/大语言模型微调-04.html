CHAPTER 02

LLaMA Factory 数据文件解析

理解数据目录结构，掌握每种文件格式的用途与配置方式

2.1 数据目录结构

📁 data/
├── 📋 dataset_info.json 数据集注册表（入口）
├── 📄 alpaca.json 单轮指令跟随格式
├── 💬 sharegpt.json 多轮对话格式
├── 🧠 multilingual-thinking.json 带思维链扩展格式
├── ⚙️ task-config.json EDS 生成工具配置
└── 📁 files/ 原始文档目录
dataset_info.json 数据集注册表 · 入口文件

LLaMA Factory 数据系统的入口。不存储数据本身，而是告诉 LLaMA Factory 每个文件"是什么格式、字段叫什么名字、该怎么解析"。

⚠️ 没有在此注册的数据文件，LLaMA Factory 完全忽略。
formatting 字段取值
取值含义
alpaca（默认）Stanford Alpaca 格式，单轮指令跟随
sharegptShareGPT 多轮对话格式
multilingual-thinking带思维链的扩展多轮对话格式
 
{
  "数据集名称A": {
    "file_name": "alpaca.json",
    "formatting": "alpaca",
    "columns": {
      "prompt":   "instruction",
      "query":    "input",
      "response": "output",
      "system":   "system"
    }
  }
}
alpaca.json 单轮指令跟随 · 适用于客服问答 
[
  {
    "instruction": "问题/指令文本",
    "input":       "",         // 补充上下文，通常为空
    "output":      "完整回答",
    "system":      ""          // 系统提示，通常为空
  }
]
instruction — 核心问题或指令，模型必须根据它生成回答
input — 可为空，附加材料（如"翻译以下文本："）
output — 标准答案，即微调的学习目标
system — 可选，为该条数据指定特定系统人格
sharegpt.json 多轮对话 · 适用于上下文连贯对话 
[
  {
    "messages": [
      { "role": "system",    "content": "系统提示词" },
      { "role": "user",      "content": "用户说的话" },
      { "role": "assistant", "content": "助手回复" },
      { "role": "user",      "content": "用户追问" },
      { "role": "assistant", "content": "助手再次回复" }
    ]
  }
]
multilingual-thinking.json 带思维链 · 训练推理型模型（DeepSeek-R1 类）

ShareGPT 格式的扩展版本，专为训练具备显式推理过程的模型设计。每条数据是 JSONL 对象（换行分隔，非数组）。

{
  "reasoning_language": "English",
  "messages": [
    { "role": "system",    "content": "",        "thinking": null },
    { "role": "user",      "content": "问题",    "thinking": null },
    { "role": "assistant", "content": "完整回答", "thinking": "推理过程..." }
  ]
}
thinking — 助手在正式回答前的"内心推理过程"
reasoning_language — 控制思维链语言（允许思维用英文、回答用中文）
task-config.json EDS 生成工具配置 · 不参与 LLaMA Factory 训练 
{
  "textSplitMinLength": 2200,
  "textSplitMaxLength": 4000,
  "questionGenerationLength": 240,
  "questionMaskRemovingProbability": 60,
  "concurrencyLimit": "20",
  "multiTurnRounds": 3,
  "splitType": "markdown"
}

记录生成时的超参数，用于复现数据集的生成过程，不参与 LLaMA Factory 的训练流程。

CHAPTER 03

LLaMA Factory 微调参数解析

从参数作用、效果影响、调整策略、显存影响四个维度全面解析每一个关键参数

3.1 核心超参数

学习率

Learning Rate
默认 5e-5 LoRA 推荐 5e-4 显存无影响
核心作用

控制模型每次看到数据后，纠正错误的"步伐大小"。AdamW 优化器的初始学习率，决定了参数更新的幅度。

动画演示：下山比喻
学习率过大（震荡/发散） 学习率合适（平滑收敛） 学习率过小（几乎不动）
过大 ❌
  • 灾难性遗忘
  • 无法收敛
合适 ✅
  • 稳定收敛
  • 知识保留
过小 ❌
  • 局部最优
  • 训练极慢
  • 过拟合风险

训练轮数

Epochs
默认 3.0 推荐 2-6 显存无影响 线性增加时间
核心作用

决定模型把数据集从头到尾完整学习多少遍。过少导致欠拟合，过多导致过拟合。

动画演示：拟合曲线变化
欠拟合（Epoch 过少） 好拟合（Epoch 合适） 过拟合（Epoch 过多） 三条曲线叠加对比
数据量小时可设到 6 轮，但要密切观察验证集损失曲线，防止过拟合。

计算类型

Compute dtype
推荐 bf16 显存减半 速度提升数倍
动画演示：格式对比
FP32
32 位
精度最高，显存最大，速度最慢
FP16
16 位
数值范围窄，容易溢出崩溃
推荐
BF16
16 位
业界标配，范围等同 FP32，训练极稳定
RTX 30/40 系列、A100/H100：直接选 bf16，这是最优解。
RTX 20 系列、V100：选 fp16（不支持 bf16 硬件加速）。

3.2 基础参数

最大梯度范数

Max Gradient Norm
默认 1.0 显存无影响

梯度裁剪的阈值。就像给方向盘加了一个"最大转向限位器"——防止异常数据造成梯度爆炸，保证训练稳定性。

默认值 1.0 适用于 99% 的情况，只有训练 Loss 大幅波动时才需要调小。

最大样本数

Max Samples
显存无影响

限制模型从数据集中读取的最大条数，用于控制数据规模和平衡多数据集比��。

🔬 调试阶段

设置 1000–2000，先跑通再大量训练，节省试错时间

🚀 正式训练

设为大于实际数据量的值，确保所有数据都参与训练

3.3 训练参数

决定模型"怎么吃透数据"以及"显卡能不能撑得住"的核心参数群

截断长度

Cutoff Length
显存平方级增长 ⚠️

模型一次能塞进脑子里的最大 Token 数量（上下文窗口大小）。

动画演示：注意力机制的平方级显存增长
线性增长（理想） 实际显存增长（平方级） 
uv run python scripts\stat_utils\length_cdf.py \
  --model_name_or_path E:\Data\Qwen3.5-4B \
  --dataset "[Easy Dataset] Alpaca" \
  --dataset_dir E:\Data\zsLQHGweOp6d \
  --template qwen3_5
使用统计脚本查看 Token 分布，按 95~98 分位数设置截断长度，避免过度浪费显存。

批处理大小

Batch Size
7B 模型推荐 1-8 显存线性增长

模型在一次前向+反向传播中，同时并行处理的样本数量。增大批次能提升 GPU 利用率，但直接线性增加显存占用。

动画演示：批改作业比喻
⚠️ 显存溢出时，先将批处理大小砍半，这是最直接有效的解决方案。

梯度累积

Gradient Accumulation
显存影响趋近零 增加训练时间

在显存不足时，通过累积多步梯度再更新来模拟大批处理的效果。

有效批次大小 = 单卡批处理大小 × 梯度累积步数 × 显卡数量
示例：1 × 16 × 1 = 16（有效批次）
用时间换空间：梯度累积不增加显存，但会延长训练时间。适合小显存用户实现大有效批次。

验证集比例

Validation Split Ratio
小数据集推荐 0.05-0.1 显存无影响

从训练数据中保留一部分专门用于检测过拟合的"试卷"。设为 0 意味着失去过拟合监控雷达。

数据集 100%
训练集 90%
验证集 10%
⚠️ 将验证集比例设为 0 时，只能看到训练集损失持续下降的"幻觉"。直到部署才会发现模型已经成为复读机。

学习率调节器

LR Scheduler
推荐 cosine 显存无影响

让学习率随训练进度动态变化的策略。余弦退火是大语言模型微调的绝对标准。

动画演示：余弦退火曲线
余弦退火（推荐） 线性下降（对比） 恒定不变（对比）
1
预热期：小油门慢慢启动，防止损坏预训练底层记忆
2
狂飙期：保持高学习率，快速逼近最优解区域
3
减速期：开始收油门，避免冲过最优解
4
停泊期：极小学习率精细调整，"滑行"至完美停泊点
