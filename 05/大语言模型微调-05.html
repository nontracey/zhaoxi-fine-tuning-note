LoRA 微调核心参数

LoRA 核心机制回顾

LoRA（Low-Rank Adaptation）是一种高效的参数微调方法。它的核心思想可以概括为四个关键步骤：

动画演示 LoRA 旁路机制
输入
冻结权重
A 矩阵
x
B 矩阵
+ 融合
输出
  1. 基础权重冻结：原始权重矩阵完全冻结，不再发生任何更新。
  2. 构建旁路矩阵：并联一条由两个小型矩阵（A 和 B）相乘构成的新计算通路。
  3. 信号融合输出：两路计算结果直接相加，作为最终输出。
  4. 仅训练旁路：反向传播中只更新轻量级的小型矩阵。

参数量锐减至原来的千分之一甚至万分之一，仅需导出一个轻量级"适配器"文件即可。

LoRA 秩（Rank）

LoRA 秩是定义适配器基础架构的最核心指标。它决定了"旁路通道"在被压缩到最窄时的维度大小。

动画演示 秩与信息容量
低秩 (Rank=8)
强正则化 防过拟合
适用：指令格式对齐、语气转换等模式化任务
高秩 (Rank=64+)
大容量 复杂知识
适用：医学报告解读、代码生成等硬核知识注入
调整原则：自底向上，够用即可。从 8 或 16 起步，按需递增。警惕边际效用递减——秩超过 64 后继续增大可能适得其反。
对显存的影响

LoRA 秩是对显存消耗具有决定性影响的核心参数。参数量与秩呈线性正相关，秩从 8 提升到 64，优化器状态显存开销暴增 8 倍。

LoRA 缩放系数

如果说 LoRA 秩决定了适配器的"结构大小"，那么缩放系数则完全决定了适配器在模型整体输出中"说话的音量"。

缩放倍率计算公式
缩放倍率 = 缩放系数 ÷ LoRA 秩
动画演示 缩放系数对比
设置过低
新信号被严重压制，模型对微调充耳不闻
黄金法则
缩放系数 = 2 × 秩，缩放倍率固定在 2 左右
设置过高
新信号暴力接管，灾难性遗忘
对显存的影响

无影响。缩放系数仅执行标量乘法，不增加网络层数，不产生新状态。

LoRA 随机丢弃（Dropout）

随机丢弃是一种经典的正则化技术，用于防止模型在有限训练集上"死记硬背"。它按设定概率随机"屏蔽"LoRA 矩阵的一部分输出元素。

动画演示 随机丢弃过程
丢弃率: 0.1 激活单元: 18 / 20
决策依据：数据集规模与质量。数据少（几百条）→ 开启 0.05~0.1；数据海量（十万级+）→ 保持为 0。
对显存的影响

显存影响微乎其微。但会轻微延长总训练时间——模型需要更多迭代步数才能达到相同损失。

LoRA+ 学习率比例

这是五个参数中最具学术深度的参数。默认 0 表示传统 LoRA 模式；非零值将激活全新的梯度更新策略。

核心原理：B 矩阵初始化为全零，导致训练初期 A 和 B 学习步调严重脱节。LoRA+ 将 B 矩阵的专属学习率放大 16 倍，打破收敛瓶颈。
对显存的影响

无显存消耗。可加速收敛，实质节省约 30% 的训练时间。

新建适配器

这是一个纯粹的工程化工作流管理选项，对"增量学习"或"多阶段微调"至关重要。

未勾选

连续叠加训练

加载已有适配器，在已积累的知识基础上继续训练。适用于增量训练、断点恢复。

勾选

重新初始化

抛弃旧适配器状态，生成全新的 A 和 B 矩阵。适用于与之前任务无关的全新实验。

02

LoRA 微调扩展模块

使用 rsLoRA

rsLoRA（秩稳定 LoRA）专门解决标准 LoRA 在极高秩时的数学崩溃问题。

标准 LoRA vs rsLoRA
标准 LoRA
缩放 = 系数 ÷ 秩
→
rsLoRA
缩放 = 系数 ÷ √秩
何时必须勾选：Rank ≥ 128 时。开启后"缩放系数 = 2 × 秩"的法则不再适用，通常将缩放系数设为固定常数（16 或 32）。
对显存的影响

完全零影响。平方根运算仅在初始化时确定一次缩放常量。

使用 DoRA

DoRA（权重分解 LoRA）将预训练权重剥离为纯粹的"方向矩阵"和一维的"长度向量"，实现解耦训练。

动画演示 DoRA 权重分解
预训练权重
→
SVD 分解
→
方向矩阵
LoRA 调整
长度向量
独立训练
建议：作为首选升级方案。在极低秩（Rank=4~8）下仍能展现强大学习能力，几乎可视为标准 LoRA 的无损升级版。
对显存的影响

显存消耗轻微增加（几十 MB）。训练吞吐量下降约 10%~20%，换取性能提升。

使用 PiSSA

PiSSA 完全颠覆了 LoRA 矩阵的初始化状态。它使用 SVD 提取原始权重的"核心骨架"来初始化旁路矩阵，而非随机噪声。

标准 LoRA

随机初始化

A 矩阵随机噪声 + B 矩阵全零。旁路通道初始无任何知识结构。

PiSSA

SVD 核心骨架初始化

提取最重要特征填充 A/B 矩阵。训练第 1 秒就具备核心知识结构。

互斥限制：PiSSA 和 DoRA 都是对底层计算流的深度改造，两者互斥，不能同时使用。
对显存的影响

SVD 分解是极度消耗内存/显存的操作，可能在训练开始前就导致崩溃。正式训练后显存占用与标准 LoRA 一致。

LoRA 微调模型加载与合并

输出文件结构

微调完成后，saves 目录下会生成一系列文件。理解每个文件的作用是工程实践的基本功。

核心模型文件

分词器相关

training_loss.png - 训练损失曲线图

推理使用与模型合并

单独加载

动态加载 LoRA

同时加载基础模型和 LoRA 适配器。适合测试微调效果、多 LoRA 动态服务。

模型合并

离线合并

将 LoRA 权重与基础模型参数直接加总，生成单一标准模型文件。

  • 提升推理速度
  • 提高部署兼容性
  • 为模型量化做准备
  • 降低系统复杂度
生产最佳实践：先在全精度状态下合并，再对合并后的单一模型进行量化。这样能最大限度保留微调所学到的知识。
